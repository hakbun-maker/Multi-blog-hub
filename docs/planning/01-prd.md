# Multi Blog Hub PRD (Product Requirements Document)

> 버전: 1.0 | 날짜: 2026-03-04

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | Multi Blog Hub |
| 핵심 가치 | 하나의 허브에서 복수의 독립 블로그를 운영하고, AI가 멀티 캐릭터로 글을 작성해 자동 배포 |
| 타겟 사용자 | 파워 블로거, 콘텐츠 마케터, AI 자동화 블로거 |

---

## 2. 핵심 기능

### 기능 1: 멀티 블로그 허브 관리
- **설명**: 하나의 대시보드에서 복수의 독립 블로그 생성·관리. 사용자가 보유한 도메인(또는 서브도메인)을 연결.
- **Why**: 구글이 각 블로그를 독립 사이트로 인식 → SEO 노출 극대화
- **우선순위**: MVP 필수
- **상세**: 사용자 도메인 연결 방식 (독립 도메인 또는 서브도메인 모두 지원)

### 기능 2: AI 멀티캐릭터 글 생성 & 동시 배포
- **설명**: 키워드 하나로 서로 다른 캐릭터/톤의 글을 복수 작성해 여러 블로그에 동시 업로드
- **Why**: 동일 키워드 기반이지만 각기 다른 캐릭터로 작성 → 중복 콘텐츠 회피 + 다양한 SEO 노출
- **우선순위**: MVP 필수
- **상세**:
  - 키워드 입력 → 연관 키워드 + 형태소 생성 → 블로그 선택 + 이미지 수 선택 → AI 생성 → 편집 → 발행
  - 각 블로그마다 AI 캐릭터 커스텀 설정 (톤 + 스타일)
  - 지원 AI: Claude / GPT-4o / Gemini (사용자 API 키 연결)

### 기능 3: 스니펫 메모 시스템
- **설명**: 광고 코드, 버튼, 제휴 링크 등을 미리 저장해두고 에디터에서 원클릭으로 삽입
- **Why**: 반복 삽입 작업 제거 → 글 작성 속도 향상
- **우선순위**: MVP 필수
- **상세**: 블로그별 스니펫 저장 + 전역 스니펫 지원

### 기능 4: 내부 자동화 스케줄러
- **설명**: Blog Hub 내부에서 모든 자동화 설정 완결. 대상 블로그, 발행 시간, 개수, 이미지 수를 자율적으로 설정.
- **Why**: 외부 도구 없이 완전 자동화 운영 가능
- **우선순위**: MVP 필수
- **상세**:
  - 키워드 풀 관리 (내부 키워드 목록)
  - 구간별 예약 발행 (주/월 단위 반복)
  - 자동화 실행 로그
  - 외부 연동 선택사항: Notion, Google Sheets (키워드 소스)

### 기능 5: 블로그 간 크로스링킹 & AdSense 최적화
- **설명**: 블로그끼리 자동 상호 링크 + 광고 노출 위치/형태 커스터마이징
- **Why**: 내부 링크 구조로 SEO 강화 + AdSense 수익 극대화
- **우선순위**: MVP 필수

### 기능 6: SEO 키워드 탐색기
- **설명**: 구글 SEO에 적합한 최신/시즌성 키워드 탐색
- **Why**: 콘텐츠 기획 단계에서 트렌드 키워드 발굴
- **우선순위**: MVP 포함 권장

### 기능 7: 외부 블로그 플랫폼 연동
- **설명**: WordPress, 아임웹 등 외부 블로그 플랫폼 연결 및 발행
- **Why**: 기존에 운영 중인 블로그 통합 관리
- **우선순위**: v2 (MVP 이후)

## 3. 사용자 스토리

| 스토리 | 우선순위 |
|--------|---------|
| 파워 블로거로서, 여러 블로그를 하나의 화면에서 관리하고 싶다. 매번 각 블로그에 로그인하는 게 번거롭기 때문에. | HIGH |
| 콘텐츠 마케터로서, 하나의 키워드로 다른 캐릭터의 글 5개를 동시에 만들고 싶다. 콘텐츠 볼륨을 빠르게 늘리기 위해. | HIGH |
| 블로거로서, 광고 코드를 매번 복붙하지 않고 버튼 하나로 삽입하고 싶다. 반복 작업이 너무 많기 때문에. | HIGH |
| 운영자로서, 매주 월요일 9시에 특정 블로그에 자동으로 글이 발행되도록 설정하고 싶다. | HIGH |
| SEO 담당자로서, 지금 트렌딩 중인 키워드를 빠르게 찾아 콘텐츠에 반영하고 싶다. | MEDIUM |
| WordPress 블로그 운영자로서, 기존 블로그도 Hub에서 함께 관리하고 싶다. | LOW (v2) |

## 4. 제약 조건

| 항목 | 내용 |
|------|------|
| AI API 비용 | 사용자가 본인 API 키 사용 → 서비스 비용 부담 없음 |
| 구글 정책 | 중복 콘텐츠 회피 위해 AI 캐릭터 다양성 필수 |
| 도메인 관리 | 사용자 도메인 연결 방식 → 인프라 복잡도 낮춤 |
| 외부 블로그 연동 | WordPress/아임웹 API 제한에 따라 기능 범위 달라질 수 있음 |

## 5. MVP 범위 (Phase 1)

### 포함
- [ ] 회원가입/로그인 (Supabase Auth)
- [ ] 대시보드 (통계 + 최신글 + 수익 현황)
- [ ] 블로그 관리 (생성, 도메인 연결, AI 캐릭터 설정)
- [ ] 글 작성 (AI 생성 모드 + 직접 작성 모드)
- [ ] 스니펫 메모 시스템
- [ ] 내부 스케줄러 (키워드 풀 + 예약 발행)
- [ ] AdSense/광고 관리
- [ ] SEO 키워드 탐색기 (기본)

### MVP 이후 (v2)
- [ ] 외부 블로그 플랫폼 연동 (WordPress, 아임웹)
- [ ] Notion/Google Sheets 연동 (외부 키워드 소스)
- [ ] 크로스링킹 자동화 알고리즘
- [ ] 고급 통계/수익 분석

---

## 6. 핵심 차별점

1. **SEO 분산 전략**: 복수 독립 도메인 → 구글 노출 극대화
2. **AI 멀티페르소나**: 같은 키워드도 각기 다른 캐릭터 글 → 중복 콘텐츠 회피
3. **자체 완결형 자동화**: 외부 도구 없이 Hub 내부에서 모든 자동화 완결
4. **멀티 AI 지원**: Claude, GPT-4o, Gemini 중 선택 (사용자 API 키)
